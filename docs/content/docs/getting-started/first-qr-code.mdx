## Save as PNG

```bash
pixcli qr generate \
  --key "+5511999999999" \
  --amount 25.00 \
  --name "LOJA EXEMPLO" \
  --city "SAO PAULO" \
  --output qrcode.png \
  --size 15
```

This creates a `qrcode.png` file you can print, embed in invoices, or share.

## Without an Amount (Open Value)

```bash
pixcli qr generate --key "email@example.com"
```

This generates a static QR code where the payer decides the amount.

## Decode a QR Code

Got a BR Code payload? Decode it to see what's inside:

```bash
pixcli qr decode "00020126580014br.gov.bcb.pix0136+5511999999999..."
```

Output:

```json
{
  "format": "01",
  "key": "+5511999999999",
  "amount": "25.00",
  "name": "LOJA EXEMPLO",
  "city": "SAO PAULO",
  "txid": "***"
}
```

## Understanding BR Code

The QR code contains an EMV-format payload called a **BR Code**. This is the standard defined by the Brazilian Central Bank for Pix payments. It encodes:

- **Pix key** ‚ÄĒ the recipient's identifier (CPF, CNPJ, email, phone, or EVP/random key)
- **Amount** ‚ÄĒ optional for static QR codes
- **Merchant name** ‚ÄĒ up to 25 characters
- **Merchant city** ‚ÄĒ up to 15 characters
- **Transaction ID** ‚ÄĒ optional identifier
- **CRC16 checksum** ‚ÄĒ integrity verification

## Static vs Dynamic QR Codes

| Feature | Static | Dynamic |
|---------|--------|---------|
| Generated offline | ‚úÖ | ‚ĚĆ (requires API) |
| Fixed amount | Optional | Required |
| Reusable | ‚úÖ | Single-use |
| Tracks payment | ‚ĚĆ | ‚úÖ |
| Expiry | None | Configurable |

Static QR codes are generated locally with `pixcli qr generate`. Dynamic QR codes are created as part of a **charge** via the API ‚ÄĒ see [pixcli charge](/docs/cli/charge).
